# Новые правила работы над проектом «Акцент» B2B Industrial Landing Page

## 2. Code Style Rules (обновлённые и интегрированные)

- **Весь код** — статичный HTML/CSS/JS в **одном файле** (стили в `<style>`, скрипты в `<script>`).  
  При переходе на Tailwind/React — отдельный файл, но текущий артефакт остаётся single-file.

- **Шрифты:**
  - Inter: через Google Fonts (`wght@300..700`)
  - IBM Plex Mono: отдельным запросом (только для артикулов, таблиц, кодов, технических данных)

- **ВСЕГДА** использовать CSS-переменные из `design-tokens.md` / tokens.css.  
  **Никаких** hardcoded hex, px, цветов, отступов.

- **Изображения:** `loading="lazy"`, `decoding="async"`, `alt` обязателен, `width` и `height` заданы явно.

- **Семантический HTML:** `header`, `nav`, `main`, `section`, `footer`, `article`.  
  **Одна единственная** `<h1>` на странице.

- **JS-анимации:** только через `IntersectionObserver`.  
  **Запрещено** `setInterval` для анимаций.

- **Хранилище:** НЕ использовать `localStorage` и `sessionStorage` (ограничения iframe).

- **Внешние ссылки:** `target="_blank" rel="noopener noreferrer"`

- **Форма BOM:**
  - Использовать EmailJS (CDN) или Formspree
  - Без серверного кода в артефакте
  - Поддерживаемые форматы: `.xls`, `.xlsx`, `.csv`, `.pdf`
  - Максимальный размер: 10 МБ

- **Комментарии в коде:** только по делу, на русском языке.

## 3. Основные правила проекта

1. Технический долг версии 1.0 — полностью убрать.
2. Гибкость: разрешены React, Tailwind, Canvas, Framer Motion, mouse-grid и purposeful анимации.
3. Один артефакт за раз — полностью рабочий.
4. Главная цель — **2–3 сильных дизайн-направления**.
5. Тон: экспертный, инженерный, деловой (только факты, цифры, сроки, % экономии).
6. Если не нравится результат — говорим «стоп» или «переделай».
